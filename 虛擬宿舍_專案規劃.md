# 虛擬宿舍(暫名)專案規劃

## 專案概念
四人共用一個線上小宿舍系統,各自有代表自己的小人物,狀態即時同步。
結合輪值提醒、留言板、共養寵物,讓室友即使不在同一個空間也能感受到彼此的存在與互動。

**重要前提**:這不是單人使用的網頁小工具,而是四人即時連動的多人系統。四台裝置各自登入後,能立即看到彼此的狀態變化(小人物位置、留言、寵物互動等)。

---

## 技術架構

- **前端**:HTML/CSS/JS,PWA(Progressive Web App)
- **部署**:GitHub Pages
- **後端**:Firebase
  - Firestore:即時資料庫,存放使用者狀態、輪值、留言、寵物資料
  - Firebase Auth:登入驗證
- **裝置**:四人各自於手機瀏覽器開啟,「加入主畫面」後體驗接近原生 App
- **登入方式**:選名字 + 簡單 PIN 碼(僅供 4 人內部使用,不需要複雜的帳號驗證系統)

> 補充:未來若確定系統穩定好用,可考慮用 Capacitor 包裝後上架 App Store。目前先不做這部分,登入機制屆時需升級為 Email 或 Sign in with Apple。

## 資料結構草案(Firestore Collections)

### `users`
```json
{
  "uid": "string",
  "name": "string",
  "pin": "string",
  "avatarStyle": "string",
  "currentRoom": "living_room | room1 | room2 | room3 | room4",
  "status": "online | offline | busy | sleeping",
  "mood": "string (optional emoji or short text)",
  "lastActiveAt": "timestamp"
}
```

### `duties`
```json
{
  "id": "string",
  "cycleType": "daily | weekly",
  "taskName": "string (例如:倒垃圾、掃地)",
  "assignedUid": "string",
  "date": "date",
  "isCompleted": "boolean"
}
```

### `messages`
```json
{
  "id": "string",
  "authorUid": "string",
  "content": "string",
  "createdAt": "timestamp"
}
```

### `pet`
```json
{
  "id": "string (單一寵物文件或每人共養一隻)",
  "name": "string",
  "hunger": "number (0-100)",
  "moodValue": "number (0-100)",
  "lastInteractionAt": "timestamp",
  "interactionLog": [
    { "uid": "string", "action": "feed | play", "timestamp": "timestamp" }
  ]
}
```

## 功能模組(三線並行,各做基礎版)

### 1. 地圖與小人物
- 一張公寓場景圖(客廳 + 4 間房)
- 四個小人物,可移動、進出房間
- 小人物狀態即時反映:在線/離線/忙碌/在睡覺
- 其他室友的小人物位置即時同步顯示

### 2. 輪值提醒 + 留言板
- 首頁顯示「今天/本週輪到誰」
- 可設定輪值週期與任務內容,標記完成狀態
- 留言板:室友互相留言、公告事項,顯示留言者與時間

### 3. 共養寵物
- 顯示飢餓值、心情值
- 點擊可餵食/互動,數值即時更新並同步給所有人看
- 長期沒人理會,數值會下降(製造大家想上線互動的誘因)

## 開發順序建議

1. Firebase 專案建置(設定 Auth + Firestore 資料結構)
2. 登入畫面(選名字 + PIN 碼)
3. 地圖與小人物移動(先用假資料測試畫面)
4. 接上 Firestore 即時同步(讓四人真正互看彼此狀態)
5. 輪值提醒 + 留言板功能串接
6. 寵物系統串接
7. 細節美化(小人物造型、動畫效果、通知提示)
